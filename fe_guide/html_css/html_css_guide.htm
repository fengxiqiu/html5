前言

§ 1

本文档为前端团队HTML / CSS编码规范，旨在规范前端页面的输出代码，增强前端编码的一致性，扩展性，可移植性。目的在于优化项目和流程，提高团队的协作开发效率。

本文档基于HTML5 / CSS3构建，请采用现代浏览器进行浏览。

HTML书写规范

§ 3
  1. 文档中尽量不要出现&nbsp;,使用CSS控制缩进。
  2. 文档中尽量不使用行内样式。
  3. 为<img/>标签加上alt属性。
  4. 为已知宽高的<img/>标签在CSS中设置宽高属性。
  5. 为<label/>标签添加for属性。
  6. 确保一个ID只在文档中出现一次。
  7. 使用四个空格的Tab来缩进子节点。
  8. 当页面上有动态数字、文字时，请为其包裹上一个标签。
  9. 标签有需要应用的数据时，为其添加data-*前缀的属性。
  10. 使用转义符输出HTML特殊字符。

&gt; > 大于号
&amp; & 连接符
&quot; " 引号

HTML标签使用

§ 4
合理使用语义化标签

在为文档设计HTML标签的时候，都为每个标签附上了它的语义和用途。在标签的使用上，我们虽然没有硬性的要求，希望开发者对常用的HTML标签有足够的认识和理解，并合理运用。

<form/>标签的使用

使用异步提交数据时，避免存在无用的<form/>标签。

<h1/>标签的使用

在文档中只可能出现一次<h1/>标签。

<img/>标签及CSS背景的使用

何时使用<img/>标签？

页面中由后端程序获取的图片使用<img/>标签，如用户头像，用户上传图片等具有上下文意义的图片。其他情况下使用CSS背景图片。

<label/>标签的使用

标签<label/>与<input/>配合使用，目的是为了用户点击文字可聚焦到输入框。

行内元素与块级元素

内联元素不允许包含块级元素，不允许空的<table/>, <ul/>, <ol/>标签，<p/>标签不允许包含任何块级元素。

CSS书写规范

§ 5
书写格式

最多使用不超过 4 个选择器书写。

尽量使用简写样式，例如：border:1px solid #000;。 get css shorthand details.

定义链接使用正确的顺序。例如：:link,:visited,:hover,:active。

一个属性写在一行。

.selector {
    property1: value1;
    property2: value2;
}

多于2个的选择器使用同样的属性时，每个各占一行。

.selector1,
.selector2,
.selector3,
...
.selectorN {
    property: value;
}
避免使用低效的选择器
body > * {...}
div#ID {...}
div#ID > h3 {...}
#ID span.class a {...}
div:last-child {...}
书写顺序

属性的书写顺序，统一按照 “显示属性 – 自身属性 – 文本属性 – 其他属性(hack)”，方便阅读、维护。

//显示属性                //自身属性                //文本属性
display                  width                    color
list-style               height                   font
position                 margin                   text-decoration
float                    padding                  text-align
clear                    border                   vertical-align
z-index                  background               white-space
                                                  other text
                                                  content
浏览器兼容

兼容多个浏览器属性时，标准属性写在最后。

-webkit-border-radius: 5px;
   -moz-border-radius: 5px;
    -ms-border-radius: 5px;
     -o-border-radius: 5px;
        border-radius: 5px;
使用以下形式区别浏览器
//区别属性
IE6                       _property: value;
IE6/7                     *property: value;
IE8                       property:value\0;
IE6/7/8/9                 property:value/9;

//区别规则
IE6                       * html selector { … }
IE7                       *:first-child+html selector { … }
not IE6                   html>body selector { … }
firefox only              @-moz-document url-prefix() { … }
saf3+/chrome1+            @media all and (-webkit-min-device-pixel-ratio:0) { … }
opera only	          @media all and (-webkit-min-device-pixel-ratio:10000),
                          not all and (-webkit-min-device-pixel-ratio:0) { … }
iPhone/mobile webkit	  @media screen and (max-device-width: 480px) { … }
使用伪类或者Overflow来清除浮动

通过触发元素的 BFC 达到清除浮动的目的。

.clearfix { *zoom: 1; }
.clearfix:before, 
.clearfix:after {
    display: table;
    content: "";
}
.clearfix:after { clear: both; }
关于z-index的属性说明

使用到z-index属性的地方统一使用10000开始递增。

它的子节点使用到z-index属性的地方统一使用1000开始递增。

它的孙子节点使用到z-index属性的地方统一使用100开始递增。

关于中文字体的声明

不建议使用字体的中文名称进行字体声明。如：font-family:"宋体"，因为当 CSS 编码设置不当的时候，会导致页面加载后CSS中的中文字体乱码。

建议使用Unicode字符来声明中文字体。常见的中文字体的编码如下：

新细明体 \65B0\7EC6\660E\4F53 细明体 \7EC6\660E\4F53
标楷体 \6807\6977\4F53 黑体 \9ED1\4F53
宋体 \5B8B\4F53 新宋体 \65B0\5B8B\4F53
仿宋 \4EFF\5B8B 楷体 \6977\4F53
仿宋_GB2312 \4EFF\5B8B_GB2312 楷体_GB2312 \6977\4F53_GB2312
微软正黑体 \5FAE\x8F6F\6B63\9ED1\4F53 微软雅黑 \5FAE\8F6F\96C5\9ED1

我们可以使用escape方法获取中文字体的Unicode编码，如escape('宋体')输出"%u5B8B%u4F53"，将%u替换成\即可。

CSS选择器命名

§ 6
命名规则

名称含义明确，尽可能简写，使用小写字母，两个单词之间使用 “-” 链接。如：.win-pop

ID选择器用于标识具体模块，class选择器用于某一类型。

尽量按照功能来命名，不按表现来命名。

//不建议使用与表现挂钩的命名
.w360 { width:360px; }
推荐使用的选择器命名
表示状态       .on, .active, .selected, .hili
表示位置       .first, .last, .main, .side
表示结构       .hd, .bd, .ft, .col, .section
通用元素       .tb, .frm, .nav, .list, .item, .tag, .pic, .info

#header 头部 .submenu 子菜单
#footer 底部 .nav 导航
#container 主体内容 .copyright 版权
.inner 容器内部 .main 主内容区
.aside 侧栏区 .content 内容区
.extra 扩展区

CSS命名约定(建议)

§ 7
什么是命名约定?
为了更好地宣告某个dom节点含有js相关操作，需要额外约定一些class类，该class类不包含有样式定义，纯粹为了宣告dom节点含有js操作。
命名约定规则
名称含义明确，尽可能描述清楚功能，尽可能简写；
两个单词之间使用 “-” 链接。如：.js-close-win；
关于命名约定class名
js-close-win 以js开头宣告节点含有js操作, close-win后半部分为功能说明描述

CSS精灵

§ 8
关于拼合图片的文件名
all_bg 拼合各种常用背景图片
btn_bg 拼合各种滑动门按钮背景图
x_bg 拼合横向平铺的背景图
y_bg 拼合竖向平铺的背景图
icon_bg 拼合各类图标的背景图
图片格式

普通图片统一使用png8格式。

需要半透明的图片使用png24格式。

背景颜色复杂的，使用JPG格式。

横向平铺的图片宽度不少于8px，竖向平铺的图片高度不少于8px。

HTML, CSS模块化

§ 9
什么是HTML,CSS模块化？

在HTML文档，模块结构功能相近的，可组合、分解和更换的单元。

为什么要HTML,CSS模块化？

提高代码重用率。

提高开发效率、减少沟通成本。

降低耦合。

提高页面容错。

减少Bug定位时间和Fixed成本。

如何实现HTML,CSS模块化？

在HTML文档，主要通过多类名来实现模块化设计。

在基类中定义默认或继承样式，在扩展类中定义具体样式。

<!-- module start -->
<div class="mod extend extra">
    ......
</div>
<!-- module end -->

代码注释

§ 10

合理的代码注释，良好的代码注释习惯，能够提高文档的可读性和维护性。

HTML代码注释

在结构或者模块的前后声明注释。

<!-- module start -->
<div class="mod mod-name">
    ......
<div/>
<!-- module end -->

<!-- header start -->
<div class="header">
    ......
<div/>
<!-- header end -->
CSS代码注释

在文档的顶部声明该文档的项目名称，维护者，日期，文档作用。

/*
 *  Project : Front to End Developer Guide
 *  Date    : 2016.07.01
 *  Author  : Tom Feng
 *  Declare : Built style for front to end developer guide
 */

按功能模块注释，在模块的结尾空一行表示结束。

/* news list */
.list li {...}
.list li a {...}

/* gallery list */
.gallery li {...}
.gallery li a {...}

邮件模版

§ 11
邮件模版规范
  1. 使用HTML5文档声明，因为能够向前兼容。
  2. 使用table布局，并充分利用表格的私有属性。如：width, height, bgcolor, align, valign。
  3. 使用行内样式。不在文档中使用<style/>标签来定义主要样式。
  4. 在<td/>标签上而不是<tr/>标签定义样式，因为有些客户端会过滤<tr/>标签上的属性。
  5. 使用<img/>来引用图片，务必定义alt属性。不使用CSS定义背景图片。
  6. 可以使用<div/>, <span/>标签来处理文档细节。尽量避免使用<p/>，因为不易清除其在不同浏览器下的默认样式。
  7. 禁止使用position, float属性。
